# Style System

Choose a visual system from the content and audience.

## Swiss System

Use for AI, technology, engineering, data, education policy, institutional strategy, and high-clarity arguments.

Traits:

- Sans-serif typography.
- 12-column grid.
- One anchor color.
- Straight edges.
- Hairline dividers.
- Large thin titles, stronger small labels.
- Strong asymmetry and whitespace.
- Diagrams, matrices, timelines, and KPI pages.

Avoid:

- Gradients.
- Shadows.
- Soft rounded cards.
- Multiple accent colors.
- Decorative icons.

Good page types:

- split statement
- before / after
- capability map
- roadmap
- matrix
- data hero
- system diagram

## Magazine System

Use for human stories, industry observation, culture, narrative talks, and image-led presentations.

Traits:

- Serif or editorial display titles.
- Large photography.
- Pull quotes.
- Strong opening spread.
- Editorial captions.
- Warm or paper-like backgrounds when appropriate.
- Fewer but more memorable pages.

Avoid:

- Too many small cards.
- Generic stock inspiration images.
- Overly corporate diagrams.

Good page types:

- full-bleed cover
- quote + image
- editorial spread
- image grid
- story timeline
- closing statement

## Executive System

Use for leadership reports, consulting-style communication, board updates, operations reviews, and formal internal decks.

Traits:

- Neutral palette.
- Dense but organized information.
- Conservative typography.
- Clear findings first.
- Tables, issue trees, action plans, and decision matrices.
- Minimal decorative imagery.

Avoid:

- Motion-heavy layouts.
- Highly experimental type.
- Over-designed covers that reduce clarity.

Good page types:

- executive summary
- decision frame
- risk matrix
- milestone plan
- operating model
- next actions

## Courseware System

Use for lectures, workshops, training, and explanatory decks.

Traits:

- Clear learning path.
- Repeated section rhythm.
- Examples and practice prompts.
- Diagrams that reveal structure.
- Bigger text than report decks.
- Moderate information density.

Avoid:

- Pure lecture-note walls.
- Decorative images that do not teach.
- Too many frameworks on one page.

Good page types:

- learning objective
- concept explanation
- example
- worked process
- comparison
- practice
- recap
